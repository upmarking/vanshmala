## 2024-04-11 - [Memoizing Highly Recursive Components]
**Learning:** The family tree renders deeply nested structures using a recursive `TreeNode` component, which includes heavy UI components (like `CoupleCard`, `SinglePersonCard`). If the parent component's state updates (e.g. opening a dialog via `handleAddRelative` or `handleViewProfile`), the lack of memoization causes the entire recursive tree to needlessly re-render, creating a severe performance bottleneck for large family trees.
**Action:** When working with recursive tree-like UI components, always proactively wrap them in `React.memo` and wrap their passed-down event handlers in `useCallback` to prevent cascading re-renders. Ensure that `.displayName` is explicitly defined for these components to satisfy React/ESLint rules and improve DevTools experience.
