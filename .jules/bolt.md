## 2024-05-24 - [React.memo on highly recursive TreeNode components]
**Learning:** The family tree builder uses highly recursive visual components like `TreeNode` (`src/components/family-tree/TreeNode.tsx`). State updates in the application (e.g. expanding/collapsing nodes, or when other sibling state updates) caused cascading re-renders down the recursive tree which is extremely expensive for large family trees. Wrapping the `TreeNode` in `React.memo()` alone is not enough; any handlers passed to it (like `handleAddRelative` and `handleViewProfile` in `FamilyTree.tsx` and `PublicTree.tsx`) must also be wrapped in `useCallback()` to provide stable references and prevent breaking the memoization.
**Action:** Always wrap highly recursive visual components like `TreeNode` in `React.memo()` and ensure that any functions passed as props to them from parent components are wrapped in `useCallback()`.
