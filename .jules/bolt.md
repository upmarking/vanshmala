## 2025-02-12 - [TreeNode Render Optimization]
**Learning:** The family tree visualizer heavily uses a `TreeNode` component (`src/components/family-tree/TreeNode.tsx`) which renders recursively for each family member in the tree. Because the tree can grow large and contains animations and multiple SVGs per child, lack of memoization causes significant performance issues due to expensive cascading re-renders during state updates. Memory indicates highly recursive visual components like `TreeNode` must be wrapped in `React.memo()` and their passed handlers wrapped in `useCallback()` to prevent expensive cascading re-renders.
**Action:** Wrap `TreeNode` with `React.memo`, wrap `SinglePersonCard` and `CoupleCard` with `React.memo` and wrap `handleAddRelative` and `handleViewProfile` in `src/pages/FamilyTree.tsx` and `src/pages/PublicTree.tsx` with `useCallback`.
